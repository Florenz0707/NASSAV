# 变更

## 前端变更

1. 资源卡更新，调整下载按钮，当未下载时，显示重点色，可点击，否则是灰色不可点击
2. 从资源详情页回退时，保留页码状态（同样适用于女优页和标签页）
3. 更新刷新的动作，支持多选：刷新m3u8，刷新元数据，重新翻译文本（需要后端同步更新API）
4. 下载页的正在下载任务的标题有bug，有时已经得到了响应，却不应用，考虑维护一个哈希表缓存（avid: title）
5. 增加批量添加功能
6. 批量下载的超时应该根据任务数而延长

## 后端变更
1. 支持刷新API（刷新m3u8，刷新元数据，重新翻译文本），可以在原有API上增加参数，也可以分离成新的API
2. 目前的`task_status`只显示一个活跃中任务和一个排队中任务，无论有多少任务正在排队，都只显示一个（无论是`GET /nassav/api/tasks/queue/status`还是`websocket`都是如此）
